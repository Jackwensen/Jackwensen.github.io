CS50笔记 0-4

入门两年多后重新过一遍基础，有些很简单的就不记录了。

week 0 Scratch

  • 把计算机看作一个解决问题的黑箱。

  • 计算机使用二进制存储、传输信息，本质上是因为计算机由无数个晶体管构成，晶体管只有 开/关 两种形式。

  • 计算机显示文字则使用ASCII（后随着普及，发展到Unicode码）码进行映射，每个字符甚至emoji都用二进制表示。

  • 图、视频、音频都是通过二进制存储，涉及到颜色最常见的是RGB，255 255 255。

    • 像素->图片->视频以及音频，以及各种格式的文件，都是基于人们一致认同的组织二进制比特的标准，来进行展示的。

week 1 C

  • C语言通过编译将C代码编译成机器可懂的二进制文件，通过./执行而打开编辑器看到乱码是因为编辑器会对二进制数字进行类似ASCII的映射。
  • C语言代码需要包含用到函数的头文件，比如stdio.h和本课程常用的cs50.h等。
  • Linux常用的命令如ls rm mv mkdir cd等。
  • 基本变量bool，char %c, double float %f, int %i, long %l, string %s。
  • 在main后出现的函数需要在main前进行声明。
  • 变量使用二进制存储，bit长度有限，所以会有溢出和精度不够的问题。

week 2 Arrays

  • 编译C语言的编译器相关问题。如GCC clang make cmake等。简单理解：gcc和clang是两个不同的编译器，而make和cmake是搭配编译器来方便用户进行多文件编译而发明的工具。
  • 编译过程：预处理，编译，汇编，链接。
  • 存储：bool&char 1字节，double long 8字节，float int 4字节。
  • char型变量既可以以字符输出，又可以以数字输出。
  • string型可以看作char型数组。
  • 命令行参数int main(int argc, string argv[])，其中argc是参数个数，argv[]是参数数组，return0或者1，可通过$?查看。
  • 由于char变量使用ASCII等编码形式，可以通过加密算法保护传输的数据。

week 3 Algorithms

  • 大O，大Ω，一个平均复杂度，一个最快情况下复杂度

  • 搜索算法：线性搜索$O(n^2$) $\Omega(1)$，二分搜索$O(log_2n$) $\Omega(1)$，二分搜索需要提前排序

  • 结构体

  • 排序算法：选择排序$O(n^2)$ ，冒泡排序$O(n^2)$ $\Omega(n)$，归并排序$$O(nlog_2n)$$ $\Omega(nlog_2n)$

  • 递归

week 4 Memory

  • 16进制即0x 0-F，相当于4位二进制，所以两个16进制位相当于一个字节。

  • C中变量、地址、指针

  • 指针

  • 字符串

    • 由于字符串变量的地址属性还会引发很多有趣的特性，比如：比较两个字符串相等，改变赋值后字符串的内容会引发原字符串的改变等。所以，复制字符串需要通过malloc方法

  • 垃圾变量：声明后没有赋值或没有分配内存。

  • 计算机内存可分为四类存储空间：machine code，global variables，heap，stack。

    • 机器代码部分是编译的程序的二进制代码。 运行程序时，该代码将加载到内存的“顶部”中。
    • 全局变量是在程序中声明的变量。
    • 堆部分是调用malloc为程序获得内存的空区。调用malloc时，则开始从顶部分配内存。⬇️
      • 调用太多malloc并且不free，会导致堆溢出（heap overflow）
    • 栈部分由程序中的函数使用，因为函数被调用，每调用一个函数就入栈以向上增长。⬆️
      • 调用太多函数并不返回会导致栈溢出（stack overflow）

  • 读取文件与传参数。
